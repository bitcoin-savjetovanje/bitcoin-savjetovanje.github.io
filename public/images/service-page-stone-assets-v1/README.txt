# Service page stone visual assets v1

These assets align `/bitcoin-konzultacija/` and `/osobni-bitcoin-standard/` with the current homepage stone-sculpture visual language.

Production WebP assets are in `public/images/service-visuals/`.
Source PNG files with alpha are in `source-transparent-png/`.

Rules:
- Use as decorative `<img>` elements with `alt=""` and `aria-hidden="true"` unless there is no nearby text equivalent.
- Keep all page text in HTML.
- Do not use `/images/medallions/` on these two service pages.
- Do not draw extra CSS stone frames, medallions, bases, numbers, or illustrations around these assets.
- Use CSS only for sizing, spacing and soft drop shadows.

Suggested roles:

Bitcoin consultation:
- consultation-hero.webp — hero visual for one deep conversation / one serious question
- consultation-scope-personal.webp, consultation-scope-family.webp, consultation-scope-business.webp — personal, family, business scope cards
- consultation-outcome-thesis.webp — clearer Bitcoin thesis
- consultation-outcome-decision.webp — clearer decision frame
- consultation-outcome-standard.webp — decide whether a full standard is needed
- consultation-flow-before.webp, consultation-flow-during.webp, consultation-flow-after.webp — before / during / after conversation

Personal Bitcoin Standard:
- standard-hero.webp — hero visual for the written standard
- standard-written-frame.webp — mockup of the written standard
- standard-layer-private-family.webp, standard-layer-business.webp, standard-layer-security.webp — three layers of the standard
- standard-process-money-obligations.webp, standard-process-debt-giving.webp, standard-process-bitcoin-money.webp, standard-process-net-worth-volatility.webp, standard-process-security-family.webp — 4–6 week process steps
- standard-business-treasury.webp — business layer section
